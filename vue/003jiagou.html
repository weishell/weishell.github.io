# 生命周期

总结

  • beforecreated：el 和 data 并未初始化
  • created:完成了 data 数据的初始化，el没有
  • beforeMount：完成了 el 和 data 初始化
  • mounted ：完成挂载
  1. beforecreate : 举个栗子：可以在这加个loading事件
  2. created ：在这结束loading，还做一些初始化，实现函数自执行
  3. mounted ： 在这发起后端请求，拿回数据，配合路由钩子做一些事情
  4. beforeDestroy： 你确认删除XX吗？可以用来解绑自定义事件，避免内存泄漏.清除定时器，解绑自定dom时间，如window scroll的监听等
  5. destroyed ：当前组件已被删除，清空相关内容
beforeCreate    组件实例刚被创建，组件属性计算之前，获取不到 props 或者 data 中的数据的，因为这些数据的初始化都在 initState 中。

created     组件实例创建完成，属性已绑定，但DOM还未生成，$el属性还不存在

beforeMount 模板编译/挂载之前 开始创建 VDOM，最后执行 mounted 钩子，并将 VDOM 渲染为真实 DOM 并且渲染数据。组件中如果有子组件的话，会递归挂载子组件，只有当所有子组件全部挂载完毕，才会执行根组件的挂载钩子。

mounted     模板编译/挂载之后

beforeUpdate    组件更新之前

updated     组件更新之后

activated   for keep-alive，组件被激活时调用

deactivated for keep-alive，组件被移除时调用

beforeDestory   组件销毁前调用

destoryed   组件销毁后调用

父子组件的生命周期：父级先created=》子组件created=》子组件mounted=》父组件mouted；当修改时，父级先beforeUpdate=>子组件beforeUpdate=》子组件updated=》父组件updated

# data

组件中 data 什么时候可以使用对象?

组件复用时所有组件实例都会共享 data，如果 data 是对象的话，就会造成一个组件修改 data 以后会影响到其他所有组件，所以需要将 data 写成函数，每次用到就调用一次函数获得新的数据。

当我们使用 new Vue() 的方式的时候，无论我们将 data 设置为对象还是函数都是可以的，因为 new Vue() 的方式是生成一个根组件，该组件不会复用，也就不存在共享 data 的情况了。

注意

注意，不应该使用箭头函数来定义 method 函数 (例如 plus: () => this.a++)。理由是箭头函数绑定了父级作用域的上下文，所以 this 将不会按照期望指向 Vue 实例，this.a 将是 undefined。
