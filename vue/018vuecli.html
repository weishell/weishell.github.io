# vuecli

1.vue-cli2安装项目

cnpm install --global vue-cli
vue init webpack my-project

2.vue-cli3新建的项目

yarn global remove vue-cli //卸载掉老版本cli
yarn global add @vue/cli
vue create hello-world//vue-cli3创建项目

# vue add

新版本的脚手架安装插件可以使用npm i ,也可也使用vue add,add还会自动进行相关配置

# vue.config.js

const port=9999;

const title="vuecli3脚手架";

module.exports={
	publicPath:"./",
	devServer:{
		port:port
	},
	configureWebpack:{
		// 向index.html注入标题
		name:title
	}
}

<%= webpackConfig.name %>
//类似jsp语法拿到这个值

可以配置启动端口，路径等等，可以查看configjs的配置

想要向 webpack 的预处理器 loader 传递选项。你可以使用 vue.config.js 中的 css.loaderOptions 选项。比如你可以这样向所有 Sass/Less 样式传入共享的全局变量：相当于再每个对应的vue文件中或者sass/less/stylus中引入了这些

// vue.config.js
module.exports = {
  css: {
    loaderOptions: {
      // 给 sass-loader 传递选项
      sass: {
        // @/ 是 src/ 的别名
        // 所以这里假设你有 `src/variables.sass` 这个文件
        // 注意：在 sass-loader v8 中，这个选项名是 "prependData"
        additionalData: `@import "~@/variables.sass"`
      },
      // 默认情况下 `sass` 选项会同时对 `sass` 和 `scss` 语法同时生效
      // 因为 `scss` 语法在内部也是由 sass-loader 处理的
      // 但是在配置 `prependData` 选项的时候
      // `scss` 语法会要求语句结尾必须有分号，`sass` 则要求必须没有分号
      // 在这种情况下，我们可以使用 `scss` 选项，对 `scss` 语法进行单独配置
      scss: {
        additionalData: `@import "~@/variables.scss";`
      },
      // 给 less-loader 传递 Less.js 相关选项
      less:{
        // http://lesscss.org/usage/#less-options-strict-units `Global Variables`
        // `primary` is global variables fields name
        globalVars: {
          primary: '#fff'
        }
      }
    }
  }
}
